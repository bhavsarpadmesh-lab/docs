# Rate Limits

| Plan | Requests/minute |
|------|----------------|
| Starter | 100 |
| Growth | 500 |
| Scale | 1000 |

Rate limit status is returned on every response:

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Requests allowed per window |
| `X-RateLimit-Remaining` | Requests remaining in current window |
| `X-RateLimit-Reset` | Unix timestamp when the window resets |

When exceeded:

```json
{
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "Too many requests. Retry after 1716595200.",
    "docs": "https://docs.inboundsage.com/errors/RATE_LIMIT_EXCEEDED"
  }
}
```

## Campaign throughput

Campaign sending uses a separate **per phone number** token bucket aligned with Meta tier limits. A campaign may move to `PAUSED` when daily limits are reached or WABA quality is `RED`.

## Idempotency

For safe retries on `POST /v1/messages/send` and `POST /v1/campaigns`, use the `Idempotency-Key` header. See [Authentication → Idempotency](/authentication#idempotency).
