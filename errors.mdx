# Errors

All errors follow a consistent format:

```json
{
  "error": {
    "code": "TEMPLATE_NOT_APPROVED",
    "message": "Template 'may_offer' is pending Meta approval.",
    "docs": "https://docs.inboundsage.com/errors/TEMPLATE_NOT_APPROVED"
  }
}
```

## Error Codes

| Code | HTTP | Description |
|------|------|-------------|
| `VALIDATION_ERROR` | 400 | Request body failed validation |
| `INVALID_AUDIENCE` | 400 | Audience contacts or segment invalid |
| `TEMPLATE_NOT_APPROVED` | 400 | Template not yet approved by Meta |
| `UNAUTHORIZED` | 401 | Missing or invalid API key / JWT |
| `INVALID_API_KEY` | 401 | API key not found or revoked |
| `INSUFFICIENT_ROLE` | 403 | Role doesn't have permission |
| `NOT_FOUND` | 404 | Resource not found |
| `CAMPAIGN_NOT_DRAFT` | 409 | Campaign must be DRAFT to activate |
| `IDEMPOTENCY_CONFLICT` | 409 | Idempotency key reused with different body |
| `WABA_NOT_CONNECTED` | 422 | No active WABA on this account |
| `PHONE_NUMBER_INACTIVE` | 422 | Phone number not registered or inactive |
| `RATE_LIMIT_EXCEEDED` | 429 | Too many requests |
| `INTERNAL_ERROR` | 500 | Unexpected server error |

## Meta delivery errors

Message and campaign sends may fail at the recipient level (e.g. Meta error `132000` template not found). Check `CampaignRecipient.error` or message status webhooks — these are not always returned as top-level API errors.

## Retries

| Status | Guidance |
|--------|----------|
| `429` | Back off using [rate limit headers](/rate-limits) |
| `500` | Retry with exponential backoff |
| `400` / `404` / `409` | Fix the request; do not retry blindly |
