# InboundSage

InboundSage is a WhatsApp Business API platform built for Indian SMBs and
developers. Connect your WhatsApp Business Account, send template messages,
run campaigns, and receive real-time delivery events — all via a clean REST API.

Built by **Spectrum Enterprises** — *Inbound marketing automation for WhatsApp*.

## Base URL

```
https://api.inboundsage.com/v1
```

All endpoints are versioned under `/v1`.

For local development:

```
http://127.0.0.1:4000/v1
```

## API conventions

| Area | Naming | Example |
|------|--------|---------|
| [Messages](/api-reference/messages/send) | **snake_case** | `waba_id`, `phone_number_id` |
| [Campaigns](/api-reference/campaigns/create) | **camelCase** | `wabaId`, `phoneNumberId`, `templateId` |
| [Templates](/concepts/templates) list/create | **camelCase** | `wabaId`, `saveAsDraft` |

Recipient numbers are always **digits without `+`** (E.164 without plus), e.g. `919876543210` for India.

Resolve IDs once with:

```bash
curl -s https://api.inboundsage.com/v1/wabas \
  -H "Authorization: Bearer is_live_YOUR_KEY"
```

Use each WABA's `id` as `waba_id` / `wabaId` and `phoneNumbers[].metaPhoneId` as `phone_number_id` / `phoneNumberId`.

## Before you start

1. [Sign up](https://app.inboundsage.com/signup) for an InboundSage account (include your WhatsApp number for optional passwordless login)
2. Connect your WhatsApp Business Account via the dashboard
3. [Create an API key](https://app.inboundsage.com/api-keys)
4. Follow the [Quickstart](/quickstart) to send your first message

## Authentication

Every request requires a Bearer token — either an **API key** (`is_live_…`) or a **JWT** from dashboard login. See [Authentication](/authentication).

## Response format

Successful responses return JSON objects. Errors use a consistent envelope:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Human-readable description",
    "docs": "https://docs.inboundsage.com/errors/VALIDATION_ERROR"
  }
}
```

Some validation responses (e.g. outside the 24-hour messaging window) may include extra fields on the JSON body; treat `error.message` as the primary user-facing text.
