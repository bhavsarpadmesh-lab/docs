# Quickstart

## 1. Get your API key

Go to [app.inboundsage.com/api-keys](https://app.inboundsage.com/api-keys),
click **New API Key**, and copy the key. It starts with `is_live_`.

<Warning>
  The API key is shown only once. Store it securely — in an environment
  variable, never in source code.
</Warning>

```bash
export TOKEN="is_live_YOUR_KEY"
```

Find your `waba_id` and `phone_number_id` in the dashboard or via `GET /v1/wabas`.

## 2. Send a message

```bash
curl -X POST https://api.inboundsage.com/v1/messages/send \
  -H "Authorization: Bearer is_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "waba_id": "YOUR_WABA_ID",
    "phone_number_id": "YOUR_PHONE_NUMBER_ID",
    "to": "919876543210",
    "type": "template",
    "template": {
      "name": "your_template_name",
      "language": "en"
    }
  }'
```

```json Response
{
  "id": "msg_uuid",
  "meta_message_id": "wamid.xxx",
  "status": "QUEUED"
}
```

<Note>
  Use the template `name` and `language` from an **APPROVED** template (`GET /v1/templates`). Recipient numbers are digits without `+` (e.g. `919876543210`).
</Note>

## 3. Receive delivery events

Register a webhook to get notified when messages are delivered or read:

```bash
curl -X POST https://api.inboundsage.com/v1/webhooks \
  -H "Authorization: Bearer is_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://yourapp.com/webhooks/inboundsage",
    "events": ["message.delivered", "message.read", "message.inbound"]
  }'
```

The response includes a `secret` shown **once** — save it for step 4.

## 4. Verify the webhook signature

Every event includes an `X-InboundSage-Signature` header.
Verify it to confirm the request came from InboundSage:

```javascript
const crypto = require('crypto');

function verifySignature(secret, rawBody, signatureHeader) {
  const expected = 'sha256=' + crypto
    .createHmac('sha256', secret)
    .update(rawBody)
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(expected),
    Buffer.from(signatureHeader)
  );
}
```

<Warning>
  Use the raw request body (before JSON parsing). Re-stringifying will break verification.
</Warning>

See [Verify signature](/api-reference/webhooks/verify-signature) for Express middleware and event envelope details.
