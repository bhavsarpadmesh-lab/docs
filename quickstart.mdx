# Quickstart

This guide walks you through authenticating, finding your WABA and template IDs, and sending a template message.

## Prerequisites

- An InboundSage account with a connected WhatsApp Business Account (WABA)
- At least one **APPROVED** message template
- `curl` and `jq` installed

## 1. Get an access token

**Option A — API key (recommended for integrations)**

Create a key at [app.inboundsage.com/api-keys](https://app.inboundsage.com/api-keys), then:

```bash
export TOKEN="is_live_your_api_key_here"
```

**Option B — JWT (dashboard login)**

```bash
export TOKEN=$(curl -s -X POST https://api.inboundsage.com/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"you@company.com","password":"your-password"}' \
  | jq -r .accessToken)
```

## 2. List your WABA and phone number

```bash
curl -s https://api.inboundsage.com/v1/wabas \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.[0] | {wabaId:.id, phoneNumberId:.phoneNumbers[0].id}'
```

Save the IDs as environment variables:

```bash
export WABA_ID="your-waba-uuid"
export PHONE_ID="your-phone-uuid"
```

## 3. Pick an approved template

```bash
curl -s https://api.inboundsage.com/v1/templates \
  -H "Authorization: Bearer $TOKEN" \
  | jq '[.items[] | select(.status=="APPROVED")][0] | {name:.name, language:.language}'
```

## 4. Send a message

```bash
curl -s -X POST https://api.inboundsage.com/v1/messages/send \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: quickstart-$(date +%s)" \
  -d '{
    "waba_id": "'"$WABA_ID"'",
    "phone_number_id": "'"$PHONE_ID"'",
    "to": "919876543210",
    "type": "template",
    "template": {
      "name": "your_template_name",
      "language": "en"
    }
  }' | jq
```

<Note>
  Replace `to` with the recipient number in international format without `+` (e.g. `919876543210` for India).
</Note>

## 5. Register a webhook (optional)

```bash
curl -s -X POST https://api.inboundsage.com/v1/webhooks \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-server.com/webhooks/inboundsage",
    "events": ["message.inbound", "message.delivered", "message.read"]
  }' | jq
```

The `secret` is returned **once** — store it securely for [signature verification](/api-reference/webhooks/verify-signature).
