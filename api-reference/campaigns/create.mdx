# Create Campaign

`POST /v1/campaigns`

Create a campaign in `DRAFT` status with an audience.
Activate it separately to start sending.

Supports [`Idempotency-Key`](/authentication#idempotency) (max 255 characters). Reusing the same key within 24 hours returns the original campaign without creating a duplicate.

## Request

```bash
curl -X POST https://api.inboundsage.com/v1/campaigns \
  -H "Authorization: Bearer is_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: campaign-launch-001" \
  -d '{
    "name": "May Offer",
    "wabaId": "YOUR_WABA_ID",
    "phoneNumberId": "YOUR_PHONE_NUMBER_ID",
    "templateId": "YOUR_TEMPLATE_ID",
    "audienceType": "INLINE",
    "contacts": [
      {
        "phone": "919876543210",
        "variables": { "1": "Padmesh", "2": "25% off" }
      }
    ]
  }'
```

## Audience types

| Type | Description |
|------|-------------|
| `INLINE` | Pass contacts directly in the request (recommended batches up to 1000) |
| `SEGMENT` | Use a saved segment from the dashboard (`segmentId`) |
| `CONTACTS` | Pass an array of contact IDs (`contactIds`) |

IMAGE/VIDEO/DOCUMENT templates may require `__headerMediaUrl` in `variables` — see [Templates](/concepts/templates).

```json
{
  "id": "5832df51-0eeb-44af-bad7-5146b07745f0",
  "name": "May Offer",
  "status": "DRAFT",

    "queued_count": 1,
    "sent_count": 0,
    "delivered_count": 0,
    "failed_count": 0
  }
}
```
