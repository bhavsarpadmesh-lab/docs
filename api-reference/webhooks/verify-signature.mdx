# Verify Signature

Every webhook request includes:

| Header | Description |
| ------ | ----------- |
| `Content-Type` | `application/json` |
| `X-InboundSage-Event` | Event type (e.g. `message.inbound`) |
| `X-InboundSage-Delivery` | Event id (same as envelope `id`) |
| `X-InboundSage-Signature` | `sha256=<hmac_hex>` of the raw body |

Verify the signature using your webhook secret:

<CodeGroup>

```javascript Node.js
const crypto = require('crypto');

app.post('/webhooks/inboundsage', (req, res) => {
  const signature = req.headers['x-inboundsage-signature'];
  const rawBody = req.rawBody; // must be raw bytes, not parsed JSON

  const expected = 'sha256=' + crypto
    .createHmac('sha256', process.env.INBOUNDSAGE_WEBHOOK_SECRET)
    .update(rawBody)
    .digest('hex');

  if (!crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(signature))) {
    return res.status(401).send('Invalid signature');
  }

  const event = req.body;
  console.log(event.type, event.data);
  res.sendStatus(200);
});
```

```python Python
import hmac, hashlib

def verify_signature(secret: str, raw_body: bytes, signature: str) -> bool:
    expected = 'sha256=' + hmac.new(
        secret.encode(), raw_body, hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(expected, signature)
```

</CodeGroup>

<Warning>
  Always verify signatures on raw bytes — before JSON parsing.
  Never verify against a re-serialized body.
</Warning>

For Express, use `express.raw({ type: 'application/json' })` and parse JSON only after verification.
