# Event Catalog

Every event follows this envelope:

```json
{
  "id": "evt_uuid",
  "type": "message.inbound",
  "created_at": "2026-05-25T02:25:29.000Z",
  "data": { }
}
```

Deliveries include headers `X-InboundSage-Event`, `X-InboundSage-Delivery`, and `X-InboundSage-Signature`.

## Message Events

| Event | When |
|-------|------|
| `message.inbound` | Customer sends a message to your WABA |
| `message.sent` | Meta accepted your outbound message |
| `message.delivered` | Delivered to customer device |
| `message.read` | Customer opened the message |
| `message.failed` | Meta rejected or failed delivery |

## Campaign Events

| Event | When |
|-------|------|
| `campaign.started` | Campaign transitions to RUNNING |
| `campaign.completed` | All recipients processed |
| `campaign.paused` | Campaign paused |

## Other Events

| Event | When |
|-------|------|
| `contact.opted_out` | Customer sent STOP |
| `template.approved` | Meta approved a template |
| `template.rejected` | Meta rejected a template |
| `waba.quality_changed` | WABA quality rating changed |
| `webhook.test` | Test delivery from the API |

## Retry Policy

Failed deliveries are retried with exponential backoff:
`5s → 30s → 5m → 30m → 2h` (up to **5 attempts**).

After repeated failures the webhook status becomes `FAILING`, then `DISABLED` after more than five consecutive delivery failures.
