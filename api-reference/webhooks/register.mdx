# Register Webhook

`POST /v1/webhooks`

Register an endpoint to receive real-time events.

```bash
curl -X POST https://api.inboundsage.com/v1/webhooks \
  -H "Authorization: Bearer is_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://yourapp.com/webhooks/inboundsage",
    "events": [
      "message.inbound",
      "message.delivered",
      "message.read",
      "campaign.completed"
    ]
  }'
```

```json Response (secret shown once)
{
  "id": "120d4ac5-...",
  "url": "https://yourapp.com/webhooks/inboundsage",
  "events": ["message.inbound", "message.delivered"],
  "secretPrefix": "is_whsec_yJV",
  "secret": "is_whsec_yJVxxxxxxxxxxxxxxxxxxxxxxx",
  "status": "ACTIVE"
}
```

<Warning>
  The `secret` is shown only once. Store it immediately — you cannot
  retrieve it again. Use it to verify incoming webhook signatures.
</Warning>

## Manage webhooks

| Method | Path | Description |
| ------ | ---- | ----------- |
| `GET` | `/v1/webhooks` | List webhooks |
| `PATCH` | `/v1/webhooks/:id` | Update `url` or `events` |
| `DELETE` | `/v1/webhooks/:id` | Remove |
| `POST` | `/v1/webhooks/:id/test` | Synchronous test delivery |
