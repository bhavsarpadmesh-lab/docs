# Send Message

`POST /v1/messages/send`

Send a WhatsApp message to a single recipient.

## Request

```bash
curl -X POST https://api.inboundsage.com/v1/messages/send \
  -H "Authorization: Bearer is_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "waba_id": "YOUR_WABA_ID",
    "phone_number_id": "YOUR_PHONE_NUMBER_ID",
    "to": "919876543210",
    "type": "template",
    "template": {
      "name": "your_template",
      "language": "en",
      "components": [
        {
          "type": "body",
          "parameters": [
            { "type": "text", "text": "Padmesh" }
          ]
        }
      ]
    }
  }'
```

<Note>
  `language` is an ISO code string (e.g. `"en"`). InboundSage forwards it to Meta as `{ "code": "en" }`. Optional `components` are passed through to the WhatsApp Cloud API.
</Note>

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `waba_id` | string | ✓ | Your WABA UUID |
| `phone_number_id` | string | ✓ | Your WhatsApp phone number ID |
| `to` | string | ✓ | Recipient digits without `+` (e.g. `919876543210`) |
| `type` | string | ✓ | `text`, `template`, `image`, `document`, `audio`, `video` |
| `template` | object | if type=template | `name`, `language`, optional `components` |
| `text` | object | if type=text | `{ "body": "Hello" }` |

Supports [Idempotency-Key](/authentication#idempotency) for safe retries.

## Response

```json
{
  "id": "fb704de9-e670-4532-ae8e-57aad4268728",
  "meta_message_id": "wamid.HBgMOTE5OTIy...",
  "status": "QUEUED"
}
```

<Note>
  Outside the 24-hour customer service window, only approved template
  messages can be sent. Free-form text is allowed within 24 hours of
  the customer's last message.
</Note>
