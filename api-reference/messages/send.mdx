# Send Message

`POST /v1/messages/send`

Send a WhatsApp message to a single recipient. Request fields use **snake_case** (unlike campaigns, which use camelCase).

Supports [Idempotency-Key](/authentication#idempotency) for safe retries.

## Request — template with contact enrichment

Use `waba_id` from `GET /v1/wabas` → `id` (InboundSage UUID, **not** Meta’s numeric WABA id). Recipient `to` is E.164 (include `+`).

```bash
curl -X POST https://api.inboundsage.com/v1/messages/send \
  -H "Authorization: Bearer is_live_YOUR_KEY" \
  -H "Idempotency-Key: send-msg-001" \
  -H "Content-Type: application/json" \
  -d '{
    "waba_id": "YOUR_WABA_UUID",
    "phone_number_id": "YOUR_PHONE_NUMBER_ID",
    "to": "+919876543210",
    "type": "template",
    "template": {
      "name": "order_confirmed",
      "language": "en",
      "components": [
        {
          "type": "body",
          "parameters": [
            { "type": "text", "text": "Padmesh" },
            { "type": "text", "text": "ORD-1234" }
          ]
        }
      ]
    },
    "contact": {
      "name": "Ramesh Sharma",
      "tags": ["checkout", "hot-lead"],
      "leadStage": "QUALIFIED",
      "segmentSource": "landing-page-form",
      "campaignSource": "Summer Promo 2026",
      "notes": "Called from landing page"
    }
  }'
```

The optional `contact` object updates CRM fields **after** the send is accepted. Enrichment is best-effort (fire-and-forget) and never blocks delivery.

## Request — authentication OTP

For approved `AUTHENTICATION` templates (e.g. `login_otp`), pass the code in **body** and **button** components:

```bash
curl -X POST https://api.inboundsage.com/v1/messages/send \
  -H "Authorization: Bearer is_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "waba_id": "YOUR_WABA_UUID",
    "phone_number_id": "YOUR_PHONE_NUMBER_ID",
    "to": "+919922227902",
    "type": "template",
    "template": {
      "name": "login_otp",
      "language": "en",
      "components": [
        {
          "type": "body",
          "parameters": [{ "type": "text", "text": "482917" }]
        },
        {
          "type": "button",
          "sub_type": "url",
          "index": "0",
          "parameters": [{ "type": "text", "text": "482917" }]
        }
      ]
    }
  }'
```

See [Templates](/concepts/templates) for category rules and creation notes.

## Request — text (24h window)

Free-form text only works within **24 hours** of the customer's last inbound message:

```json
{
  "waba_id": "YOUR_WABA_UUID",
  "phone_number_id": "YOUR_PHONE_NUMBER_ID",
  "to": "+919876543210",
  "type": "text",
  "text": { "body": "Thanks for your order — it ships today." }
}
```

Outside the window the API returns `422` with `use_template: true` — send an approved template instead.

## Contact enrichment fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Display name to set or update |
| `tags` | string[] | **Additive** — merged with existing tags (max 10, 50 chars each) |
| `leadStage` | string | Funnel stage: `NEW`, `CONTACTED`, `QUALIFIED`, `PROPOSAL`, `WON`, `LOST` |
| `segmentSource` | string | Segment, landing page, or integration label (max 100 chars) |
| `campaignSource` | string | Campaign or broadcast that sourced this contact (max 100 chars) |
| `notes` | string | Note on the contact record (max 200 chars; best-effort) |

`tags` never replace the full tag list — they only add new tags.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `waba_id` | string | ✓ | InboundSage WABA UUID (`GET /v1/wabas` → `id`) |
| `phone_number_id` | string | ✓ | Meta phone ID (`phoneNumbers[].metaPhoneId`) |
| `to` | string | ✓ | Recipient E.164 (e.g. `+919876543210`) |
| `type` | string | ✓ | `text`, `template`, `image`, `document`, `audio`, `video` |
| `template` | object | if type=template | `name`, `language`, optional `components` |
| `text` | object | if type=text | `{ "body": "Hello" }` |
| `image` / `document` / `audio` / `video` | object | if matching type | Meta media object (`link` or `id`) |
| `contact` | object | | Optional CRM enrichment (see table above) |

<Note>
  `language` is a plain string (e.g. `"en"`). InboundSage forwards it to Meta as `{ "code": "en" }`. `components` are passed through to the WhatsApp Cloud API unchanged.
</Note>

## Response

```json
{
  "id": "fb704de9-e670-4532-ae8e-57aad4268728",
  "meta_message_id": "wamid.HBgMOTE5OTIy...",
  "status": "SENT"
}
```

| Field | Description |
|-------|-------------|
| `id` | InboundSage message log UUID |
| `meta_message_id` | WhatsApp message ID (`wamid.…`) |
| `status` | `SENT` on success (or `QUEUED` briefly while processing) |

## Errors

| Situation | HTTP | Notes |
|-----------|------|-------|
| Outside 24h window (text send) | 422 | Body includes `use_template: true` |
| Contact opted out | 422 | `CONTACT_OPTED_OUT` |
| Invalid template / Meta rejection | 400 | Meta error message in `error.message` |
| Missing auth | 401 | Invalid or missing API key |

Delivery failures after accept are reported via [webhooks](/api-reference/webhooks/events).
